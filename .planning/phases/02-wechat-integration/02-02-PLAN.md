## Key Decisions to Implement

## Enterprise WeChat API Details (from RESEARCH.md)
- Endpoint: POST https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key={key}
- Payload: {"msgtype": "markdown", "markdown": {"content": "..."}}
- Response: {"errcode": 0, "errmsg": "ok"} for success
- Limits: 2048 chars max, 20 msg/sec rate limit
- Retriable errors: -1 (busy), Timeout, ConnectionError
- Non-retriable: 40013 (invalid key), 40008 (message too large)
