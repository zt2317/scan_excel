# 功能研究

## 用户角色

**主要用户：物流/仓储人员**
- 每天需要处理Excel格式的物流数据
- 需要快速通知团队异常数据
- 不熟悉编程，需要简单易用的GUI

## 功能分类

### 核心功能（必须）

**Excel文件选择**
- 浏览文件系统选择Excel文件
- 支持.xlsx和.xls格式
- 显示选中文件路径

**数据读取**
- 自动识别指定列：日期、发货人、提单号、入库未扫、出库未扫
- 支持列名映射（用户可能使用不同列名）
- 数据预览（显示前N行）

**Markdown格式化**
- 生成标准Markdown表格
- 包含表头
- 数据对齐

**企业微信推送**
- 配置webhook URL
- 推送消息到指定群组
- 发送成功/失败提示

**消息切片**
- 自动检测消息长度
- 按企业微信限制（2048字符）切片
- 连续发送多条消息

### 增强功能（可选）

**配置持久化**
- 保存webhook URL
- 保存上次选择的文件夹路径

**数据过滤**
- 只推送"入库未扫"或"出库未扫"不为空的行
- 按日期范围筛选

**发送历史**
- 记录发送时间和内容摘要
- 显示最近发送记录

### 反功能（不应包含）

- **Excel编辑**：仅读取，不修改源文件
- **多IM支持**：仅企业微信，不扩展钉钉/飞书
- **数据存储**：不保存Excel内容到本地数据库
- **复杂格式**：不支持合并单元格、图表等

## 用户体验要求

**界面简洁**
- 单窗口应用
- 步骤清晰：选文件 → 预览 → 发送

**操作反馈**
- 加载Excel时显示进度
- 发送成功/失败明确提示
- 错误信息友好（非技术语言）

**容错处理**
- Excel格式错误时给出清晰提示
- 网络失败时可重试
- 列名不匹配时允许手动映射

## 企业微信API限制

**消息长度**
- 文本消息最多2048字符
- Markdown格式占用更多字符
- 建议切片阈值：1800字符（预留格式空间）

**频率限制**
- 每应用每群最多20条/秒
- 切片发送需注意间隔

**Webhook格式**
```json
{
  "msgtype": "markdown",
  "markdown": {
    "content": "markdown内容"
  }
}
```
