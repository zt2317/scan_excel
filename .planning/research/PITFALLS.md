# 风险与陷阱研究

## 常见陷阱

### 1. 打包陷阱

**❌ 陷阱：尝试在Mac上直接生成Windows exe**
- PyInstaller不支持交叉编译
- 在Mac运行`pyinstaller --onefile app.py`只能生成Mac应用
- 必须使用Windows环境打包

**✅ 解决方案：**
- 使用GitHub Actions Windows runner
- 或本地Windows虚拟机

---

### 2. Excel解析陷阱

**❌ 陷阱：openpyxl不支持.xls格式**
- 只安装openpyxl无法读取旧版Excel
- 会抛出`InvalidFileException`

**✅ 解决方案：**
- 同时安装xlrd处理.xls
- 或安装pandas统一处理（但打包体积大）

**❌ 陷阱：列名不匹配**
- 用户Excel可能使用"提单号码"而非"提单号"
- 或"日期"列实际是"时间"

**✅ 解决方案：**
- 实现模糊匹配（如包含"提单"即匹配）
- 允许用户手动选择/映射列
- 提供预览确认

**❌ 陷阱：空值处理**
- 空单元格读取为None
- 格式化时可能出错

**✅ 解决方案：**
- 统一将None转换为空字符串"-"
- 或显示"未填写"

---

### 3. 企业微信推送陷阱

**❌ 陷阱：webhook URL泄露**
- 硬编码在代码中会导致安全问题
- webhook可能被滥用
- 提交到git会被公开

**✅ 解决方案：**
- 用户配置本地存储（配置文件或环境变量）
- 不提交到版本控制
- 添加.gitignore: `config.json`, `*.xlsx`, `*.xls`

**❌ 陷阱：频率限制**
- 切片发送过快可能触发限流
- 20条/秒的限制

**✅ 解决方案：**
- 切片间添加延迟（0.5-1秒）
- 实现重试机制（指数退避）

**❌ 陷阱：Markdown格式错误**
- 企业微信对Markdown支持有限
- 复杂格式可能无法显示

**✅ 解决方案：**
- 使用基础Markdown语法（表格、加粗）
- 避免HTML标签
- 测试验证格式

**❌ 陷阱：消息发送成功但未收到**
- webhook返回200不代表消息到达
- 可能是机器人被禁用或群组删除

**✅ 解决方案：**
- 发送后检查返回码
- 提示用户验证是否收到
- 提供测试发送功能

---

### 4. GUI陷阱

**❌ 陷阱：Tkinter界面冻结**
- 读取大Excel文件时阻塞主线程
- 界面无响应

**✅ 解决方案：**
- 使用多线程（threading）
- 显示加载状态/进度条

**❌ 陷阱：Windows/macOS字体差异**
- 字体在不同系统表现不一致

**✅ 解决方案：**
- 使用系统默认字体
- 不指定特定字体名

**❌ 陷阱：高DPI显示模糊**
- Windows高分屏可能出现模糊

**✅ 解决方案：**
- 启用DPI感知
```python
from ctypes import windll
windll.shcore.SetProcessDpiAwareness(1)
```

---

### 5. 依赖陷阱

**❌ 陷阱：打包后缺少依赖**
- PyInstaller可能漏掉某些隐藏导入
- 运行时报`ModuleNotFoundError`

**✅ 解决方案：**
- 使用`--hidden-import`显式包含
- 测试打包后的exe
- 检查build/warn-*.txt警告

**❌ 陷阱：反病毒误报**
- PyInstaller打包的exe常被误报为病毒
- 用户无法运行

**✅ 解决方案：**
- 使用--onefile模式减少误报
- 考虑代码签名（需证书）
- 向用户说明这是正常现象

---

### 6. 文件路径陷阱

**❌ 陷阱：Windows路径分隔符**
- Mac使用/，Windows使用\
- 硬编码路径会导致问题

**✅ 解决方案：**
- 使用`os.path.join()`或`pathlib`
- 不要硬编码分隔符

**❌ 陷阱：中文路径乱码**
- 打包后中文路径可能无法正确显示

**✅ 解决方案：**
- 确保源码文件使用UTF-8编码
- 在spec文件中指定编码

---

## 开发阶段应避免的陷阱

### 阶段1：核心功能开发
- **陷阱**：过早优化GUI美观度
- **避免**：先实现功能，再美化界面

### 阶段2：打包测试
- **陷阱**：只在Mac测试就发布
- **避免**：必须在真实Windows环境测试

### 阶段3：部署交付
- **陷阱**：不给用户配置说明
- **避免**：提供详细的使用文档
